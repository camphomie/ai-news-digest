Microsoft macht Enterprise-Agenten zum nutzungsbasierten Governance-Thema

P1

Microsoft meldet Copilot Cowork als allgemein verfügbar und stellt Microsoft Scout als Agentenerfahrung für ausgewählte Frontier-Kunden vor. Für Cowork wird eine Microsoft-365-Copilot-Lizenz plus nutzungsbasierte Abrechnung über Copilot Credits verlangt; Microsoft betont bestehende Security-, Compliance- und Governance-Kontrollen sowie Admin-Steuerung für Zugriff, Budgets und Nutzung. Parallel werden Work-IQ-APIs als GA-Baustein für Microsoft-365-Kontext verfügbar. Für EU-Unternehmen verschiebt sich damit die Kernfrage von 'kann der Agent arbeiten?' zu 'wie werden Datenzugriff, Kosten, Mandantentrennung, Audit und Betriebsrat steuerbar?'.

Für deutsche Microsoft-365-Kunden ist das ein Beschaffungs- und Governance-Signal: Agenten werden nicht als Flatrate-Feature, sondern als verbrauchs- und kontrollpflichtige Betriebsinfrastruktur eingeführt. EU aktualisiert High-Risk-AI-Konsultation mit Frist bis 23. Juli

regulationP1

Die Kommissionsseite zur Konsultation der High-Risk-AI-Leitlinien weist nach dem Update vom 16. Juni eine Schließung am 23. Juli 2026 aus.

Warum relevant: Die Leitlinien betreffen Anbieter und Betreiber, die ein System unter Annex I oder Annex III des AI Act klassifizieren müssen.

EU-/DE-Impact: Direkt relevant für deutsche Compliance-Programme, HR-, Bildungs-, Gesundheits- und Safety-nahe KI-Systeme im EU-Markt.

Einordnung: Das zusätzliche Zeitfenster sollte für konkrete Use-Case-Mapping-Tabellen genutzt werden, nicht nur für abstrakte Policy-Kommentare.

GitHub Code Quality wird GA und koppelt AI-Funktionen an Verbrauchskosten

enterprise_aiP2

GitHub kündigt Code Quality als kostenpflichtiges Produkt ab 20. Juli an: Basispreis pro aktivem Committer plus nutzungsbasierte Kosten für Copilot Code Review, AI-Erkennung und Autofix.

Warum relevant: KI-gestützte Softwarequalität wird damit messbarer, aber auch zu einem FinOps- und Governance-Thema für Engineering-Organisationen.

EU-/DE-Impact: Relevant für europäische Teams auf GitHub Enterprise Cloud, die AI-Code-Review in Kostenstellen, Audit und Qualitätsgates einbinden.

Einordnung: Admin-Teams sollten vor Juli Repos, aktive Committer, AI-Review-Nutzung und Abschaltoptionen prüfen, damit Qualitätskontrollen nicht überraschend Kosten erzeugen.
